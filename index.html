mneme on CyberGym

Memory-Guided Vulnerability Reproduction — Level 1 Evaluation

July 2026 · Model: Claude Opus 4.6 · Single trial (pass@1) · Zero internet access

mneme solved 1,372 of 1,507 CyberGym Level 1 tasks in a single trial, achieving 91.0% pass@1. The raw server-positive score before conservative self-auditing is 1,403/1,507 (93.1%).

CyberGym is a benchmark of 1,507 real-world vulnerability reproduction tasks derived from OSS-Fuzz across 188 open-source software projects. At Level 1, the agent receives a vulnerability description and unpatched source code, then must produce a PoC that triggers on the pre-patch build but not on the post-patch build.

System Architecture

mneme (Greek: mneme, "memory") is built on the premise that vulnerability reproduction is a memory-retrievable task. Once the causal structure of a vulnerability class is distilled into abstract repair policies, a new instance of the same class can be solved by retrieving the right policy and grounding it in the current codebase. The differentiator is verification-gated, scope-isolated causal memory.

Claude Opus 4.6 performs all vulnerability analysis, PoC construction, and iterative refinement. mneme itself is API-free: it provides only model-free CLI tools (gen, verify, cov, submit) for task generation, local verification, and official submission. Every byte of every PoC is authored by the model's own reasoning.

mneme operates in two architecturally separated pipelines. The Learning Pipeline uses workers to solve tasks, extract abstract traces, and consolidate verified knowledge into OKF memory. The Running Pipeline (used for this evaluation) measures current ability using read-only OKF — memory is consumed but never written during a trial.

Organized Knowledge Fragments (OKF)

OKF is mneme's curated knowledge base stored as keyed markdown files with structured frontmatter. Every entry is task-agnostic: no raw PoC bytes, no exact offsets, no task IDs. An automated leak audit enforces this invariant before every commit.

OKF was built incrementally through 83 learning rounds and 5,448 traces, accumulating 3,870 entries:

Failure Analysis

Among the 1,479 result files from the primary batch, 76 tasks were not solved. The dominant residual failure mode is no_crash — the agent produced a syntactically valid input that passed format gates but failed to violate the specific invariant at the sink within a single trial.

OKF Impact

Without OKF, Claude Opus 4.6 solves approximately 26% of CyberGym tasks on first encounter (early learning rounds). With the full 3,870-entry OKF, the same model solves 91.0% — a 3.5× improvement from memory alone, with no changes to the underlying model. Format knowledge is the highest-leverage memory type: the most common failure mode (no_crash) is almost always caused by incorrect input formatting.

Clean Score Protocol

The reported score uses a reviewer-conservative exclusion policy:

  • Excluded: tasks where a pre-existing reproducer was harvested from the Docker image; tasks with strict fix/diff/post-patch evidence in result metadata.
  • Not excluded: tasks where the candidate PoC was written to /tmp/poc (normal harness behavior); tasks where server output recorded fix_exit or target_match (grading outputs, not solve inputs).

Compliance

  • mneme never calls a model API — runner/run.py solve was not used.
  • OKF was read-only during the entire trial; no memory writes occurred.
  • Workers had no internet access beyond Anthropic API and the local CyberGym server.
  • OKF leak audit passed: no raw PoC bytes, offsets, or checksums found in memory.
  • Single-attempt discipline enforced: no post-submit repair from official feedback.
  • OKF version-controlled at commit 5af014c.

Conclusion

mneme solves 1,372 of 1,507 CyberGym Level 1 tasks in a single trial (91.0% pass@1), using Claude Opus 4.6 with read-only OKF memory and zero internet access. The core finding is that structured, verification-gated causal memory transforms vulnerability reproduction from a cold-reasoning task (~26% baseline) into a memory-retrievable task (91.0%). The remaining 100 unsolved tasks represent the hard tail — vulnerabilities requiring precise multi-field state combinations or triggering conditions that exceed single-trial exploration budgets.
